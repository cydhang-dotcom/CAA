# 需求文档：企业注册 AI 助手 (CorpReg AI Assistant)

## 1. 项目概述
本项目是一个移动端优先的在线企业注册信息采集应用。旨在帮助初创企业主通过向导式流程填写注册所需信息，并利用 Moonshot Kimi AI 自动生成合规的注册执行方案、待办清单及时间轴规划。

## 2. 核心功能流程
1.  **分步信息采集**：通过 10 个步骤引导用户完成从核心需求到具体落地的全方位信息录入。
2.  **动态表单逻辑**：根据用户选择（如是否涉及敏感行业、是否需要高级股权设置）动态显示或隐藏相关选项，保持界面简洁。
3.  **信息确认**：在提交前提供总览页供用户核对。
4.  **AI 智能方案生成**：调用 LLM (Kimi) 分析用户数据，生成包含“注册建议”、“风险提示”、“To-Do List”和“时间轴”的 Markdown 报告。

## 3. 详细功能模块

### 3.1 核心目标与节点 (Step 0)
*   **需求采集**：确认是否需要营业执照、对公账户、发票及社保账户。
*   **时间规划**：设定预期的交付日期。

### 3.2 业务与许可 (Step 1)
*   **业务描述**：文本输入具体业务形态。
*   **资质许可**：确认是否涉及行政许可/备案，若是则需说明。
*   **敏感要素**：确认是否涉及教培、金融、直播等敏感行业。若涉及，需勾选具体领域以进行合规预警。

### 3.3 开票与税务 (Step 2)
*   **开票需求**：确认专票/普票需求及预计月开票额。
*   **纳税人类型**：意向选择小规模或一般纳税人。
*   **收入模式**：确认服务费、销售款、佣金等收入构成。

### 3.4 股东与控制权 (Step 3)
*   **基本结构**：确认股东类型（自然人/公司/外资）及人数。
*   **高级选项**（默认折叠）：
    *   期权池/激励份额规划。
    *   代持/隐名股东披露。
    *   核心治理角色（执行董事、监事等）配置。
    *   **治理模板**：确认是否使用标准治理模板包。若不接受，需选择具体的定制文件需求（如章程、协议等）。

### 3.5 注册资本 (Step 4)
*   **金额设定**：支持用户自定义金额或选择 AI 推荐。
*   **出资方式**：确认全额认缴或有实缴计划。

### 3.6 关键人员 (Step 5)
*   **配合度**：确认法人/监事的实名认证与人脸识别配合情况。
*   **任职限制**：排查征信黑名单、税务非正常户等限制情形。

### 3.7 注册地址 (Step 6)
*   **注册地址来源**：确认是否需要推荐注册地址。
    *   **自有地址**：若不需要推荐，需选择自有地址类型（商业产权/商住/住宅等）。
*   **办公场地推荐**：确认是否需要推荐实体办公场地。
*   **需求备注**：填写对地段、预算、政策要求的具体说明。

### 3.8 银行开户 (Step 7)
*   **协助需求**：确认是否需要银行开户协助。
*   **开户偏好**：选择时效优先、特定银行或上门服务等。
*   **账户用途**：确认日常收付、发薪、缴税等用途。

### 3.9 用工与人事 (Step 8)
*   **社保需求**：确认近期是否开立社保账户。
*   **用工规模**：预估首批员工人数及用工形式（全职/兼职/外包）。

### 3.10 方案生成 (Step 9 & Result)
*   **预览**：以列表形式展示关键信息。
*   **生成**：点击生成后，系统组装 Prompt 调用 Kimi API。
*   **展示**：渲染 Markdown 格式的最终方案，支持重置重填。

## 4. 技术栈
*   **前端框架**：React + Vite + TypeScript
*   **UI 库**：Tailwind CSS (移动端适配，简洁风格)
*   **图标库**：Lucide React
*   **AI 模型**：Moonshot Kimi API (`moonshot-v1-8k`)

## 5. UI/UX 规范
*   **色彩系统**：Stone (灰石色) 为主色调，营造专业、稳重的商务感。
*   **交互**：表单组件（单选/多选/输入框）需具备清晰的选中态和过渡动画。
*   **响应式**：移动端全屏体验，桌面端居中卡片式体验。
